chrome41-input-bug

2019-06-19 
chrome41版本浏览器，vue双向绑定数据的input输入框有@input事件，输入汉字会出现拼音 例：“中” ==》 “zhong中”
        element-ui组件也会有此问题

解决办法：不在使用@input事件 使用change事件，需要全局webpack版本 ^4.30.0, vue -V ^3.6.3, webpack-cli -v ^3.3.1,  node -v ^10.13.0, npm -v ^6.4.1(没研究具体哪个依赖影响的)。
        ps: 还有同事用了win7系统 node -v ^10.13.0, npm -v ^6.4.1 发现没有这个问题，就算有@input事件都不影响 （不明白其中原因）
